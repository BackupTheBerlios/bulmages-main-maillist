>  Estoy intentando crear una nueva rama en GIT, pero tiene que estar el
> blanco. Es decir, no debe contener archivos ni estar basado en la rama
> MASTER, para albergar nuevos archivos diferentes. No lo consigo de

>
> ¿Alguna sugerencia?
>
>  Aquí mis apuntes del gran maestre Ripoll:
>
>  Creamos una nueva rama en base a la rama master.
>
> 0.) git pull
>
> 1.) git checkout --track -b master-fix origin/master
>
> --track = permite actualizar estando dentro de la nueva rama
> master-fix = nombre nueva rama
> origin/master = rama principal
>
>
> 2.) Realizamos nuestro trabajo [editar,commit]
>
>
> 3.) git rebase -i master
>
> Nos abrirá nuestro editor, mostrandonos los commits con diversas
> opciones
> pick = Coge el commit
> squash = Coge el commit, pero lo adjunta al anterior
> edit = edita el commit.
>
> Recomendado: primer commit "pick", resto "squash", logrando
> fusionarlos en un super-commit.
>
>  Volvemos a la rama master y mezclamos los cambios.
>
> 1.) git checkout master
>
> 2.) git pull
>
> 3.) git merge master-fix
>
> 4.) Solucionamos los conflictos si se diera el caso { Ver Migrando a
> GIT (III) }.
>
> 5.) git push
>
>
> * Si queremos eliminar la rama master-fix, porque ya no la
> necesitamos:
>
> 1.) git branch -d master-fix
>
