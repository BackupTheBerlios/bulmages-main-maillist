Summary: El nom d'empresa no accepta caràcters accentuats

Details: Versió: 0.12 i trunk
Instal·lació: Compilació neta a partir de les fonts del SVN.

En crear una nova empresa, el Bulmasetup no tracta bé els accents en el nom de l'empresa.

Per exemple: "AMPA Progrés" queda emmagatzemat a la base de dades com a "Sin definir".

Si entrem a la base de dades i inserim la cadena a mà (fent "
update configuracion set (valor)=('AMPA Progrés') where nombre='NombreEmpresa'\g"), aleshores en tornar a entrar al Bulmasetup i editar l'empresa existent, el nom apareix com "AMPA ProgrÃ©s".

El meu sistema usa UTF-8 per defecte, o sigui que sembla un problema de codificació de caràcters.


Follow-Ups:

Date: 2011-Jul-03 18:51
By: wgarcia

Comment:
S'introdueixen "decode" en el codi python de bulmasetup per poder entrar noms d'empresa amb caràcters de 8bit. El nom de la base de dades encara s'ha d'entrar únicament amb caràcters ASCII, es podria entrar un advertiment si l'usuari fa servir caràcters no ASCII en el nom de la base de dades.
