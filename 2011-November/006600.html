Hola. No os asusteis :)

Estoy cansado... de gastar papel para probar BulmaTPV. jeje. Así que he
creado una pequeña herramienta "ScreenTicket" que interpreta el binario
ESC/POS que le llega a la impresora de tickets y lo muestra en pantalla.
Otra ventaja es que puedes tener tantas impresoras de tickets como
quieras y no tienes que comprar impresoras físicas.

Para poderlo utilizar hay que configurar CUPS para que admita la
creación de impresoras redirigidas a un archivo.

Editar /etc/cups/cupsd.conf y añadir la siguiente directiva:

    FileDevice Yes

Guardar y reiniciciar CUPS para que los cambios surtan efecto. Luego, se
crean las impresoras de tipo "raw" (en bruto). Por ejemplo se puede
crear una impresora que redirija al archivo temporal (útil para pruebas)
"/tmp/printer1.prn" (para crear el archivo se puede usar "touch
/tmp/printer1.prn").

Se configura BulmaTPV para que use esas nuevas impresoras CUPS y se pone
en marcha el programa "ScreenTicket" con:

./screenticket --file="/tmp/printer1.prn" --title="COCINA"


Adjunto una captura de pantalla de cómo queda usando 2 impresoras.


El código fuente (GPL3+) se puede descargar desde aquí:

https://gitorious.org/screenticket


Lo he puesto en un proyecto a parte porque creo que puede servir también
para otros proyectos independientes de BulmaGes.
