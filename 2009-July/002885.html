On Friday, July 31, 2009, Xavi Drudis Ferran wrote:
> > 5) OK a la generación de la cabecera, que se llama 'local_BlI18n.h',
> > ¿o generar la cabecera con otro nombre...?
>
> No sé. M'és igual. Perquè local ?

Porque se usa de forma local, en el directorio del plugin para el cual se ha 
creado, en contraposición con la cabecera "bli18n.h" que está en bulmalib/src 
y se usa de manera global.

> > 6) No generar la cabecera, mejor que sea un fuente estático de cada
> > target.
>
> Diria que millor generar-la, no ? Això simplificaria crear plugins nous, no
> ?

Generarla permitirá mayor flexibilidad, por ejemplo con solo cambiar la macro 
de cmake podremos añadir una nueva definición de macro C++ para todos los 
plugins. Por otra parte, en los CMakeLists.txt de cada plugin se sustituyen 
todas las (feas) líneas siguientes:

	# i18n
	if(WIN32)
	ADD_DEFINITIONS("-D_(String)=BlI18n(\\\"${plugin}\\\",String)" )
	ADD_DEFINITIONS("-DN_(String,Int)=BlI18n(\\\"${plugin}\\\",String)" )
	endif(WIN32)
	if(NOT WIN32)
	ADD_DEFINITIONS("-D'_(String)=BlI18n(\"${plugin}\",String)'" )
	ADD_DEFINITIONS("-D'N_(String,Int)=BlI18n(\"${plugin}\",String)'" )
	endif(NOT WIN32)

Por esta otra línea, mucho más clarita:

	ADD_LOCAL_BLI18N_HEADER( ${plugin} )

Que suponiendo que ${plugin} tenga el valor "pluginbl_report" genera la 
cabecera siguiente:


	/* Generated file. Please do not edit! */
	/* created by MacrosBlI18n.cmake */
	#ifndef LOCAL_BLI18N
	#define LOCAL_BLI18N

	#include "bli18n.h"

	#ifndef _
	 #define _(String) BlI18n("pluginbl_report",String)
	#endif
	#ifndef N_
	 #define N_(String,Int) BlI18n("pluginbl_report",String)
	#endif

	#endif /*LOCAL_BLI18N*/
