Idea fantastica donde las existan, así matamos dos pajaros de un tiro, 
fusionando ambas dos.

Otra cosa que sería interesante sería investigar como realizar una buena 
API (tanto externa, como interna), establecer unos buenos metodos y 
documentarlos en la wiki (aviso, que la estoy migrando a un nuevo 
servidor, como ya dije, de hecho bulmages.es ya lo está). Asi pues listo 
mis propuestas.

- Qt5 y C++11

- Fusión de BulmaCont y BulmaFact

- Sistema integrado de ayuda.

- API interna (Dividir gráfico de nucleo)

- API externa (SOAP y/o XMLRPC) *CLAVE SI QUEREMOS EXPANDIRNOS*


Por mi parte, respecto al "AndroBulmaSetup" (A falta de un nombre mejor) 
lo dejé un poco aparcado, las funciones de postgres de Qt dejan que 
desear y el proceso de libpq estaba un poco aparcado, de ahí la 
necesidad de la API externa.

Saludos
