Ahhhhhhhhhhhhh, ya capto.

Pués tenemos precisamente un proyecto muy bonito (Y pensado con esa misma 
idea) que persigue justamente la idea que mencionas.

El proyecto es BulmaTPV y su esquematica es la siguiente:

	El programa está vacio en cuanto a interficie visual y se rellena mediante 
plugins (Esto significa que los plugins manejan completamente la interficie 
gráfica y además son acumulables).
	De esta forma podrias, por ejemplo, crear un plugin de creación de albaranes 
(unicamente) y meterlo en el BulmaTPV para conseguir un puesto exclusivo de 
introduccion de albaranes. Si creas un módulo de introducción de facturas 
podrias tener en BulmaTPV un puesto para solo facturas uno para solo 
albaranes o uno con los dos modulos puestos. 
	Y como el programa permite la configuración de los plugins segun la empresa y 
segun el usuario que ejecuta el programa puedes hacer versiones de programa 
personalizadas para cada usuario y empresa.

	Además BulmaTPV puede incorporar los componentes de BulmaFact con lo que 
puedes aprovechar un montón de código ya hecho.

	Igual el nombre de TPV parece una mala elección para este tipo de software. 
Pero si pensamos que el TPV no es más que un programa que sólo permite la 
introducción rápida de compras creo que el concepto se le ajusta como un 
guante.
