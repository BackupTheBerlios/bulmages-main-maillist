> Hola Freud
>
> Sobre el plugin de sincronización no se instala automaticamente porque es
> algo
> complejo de instalar. Te explico los pasos a seguir y si te apetece lo
> pones
> en el wiki:
>
> Supongamos que ya tienes dos bases de datos (una bulmafactdb y otra
> bulmacontdb)
>
> 1.- Necesitas el paquete postgresql-contrib para poder usar dblink
> $ sudo aptitude install postgresql-contrib
>
> entro como usuario postgres para manejar bases de datos aunque hay otras
> formas de hacer esto.
> $ sudo su
> # sudo postgres
>
>
> 2.- En la base de datos de bulmafact tienes que instalar el dblink.
> $ psql bulmafactdb < /usr/share/postgresql/8.3/contrib/dblink.sql
>
> 3.- En la misma base de datos tienes que instalar el parche que proporciona
> el
> plugin (la ruta puede que cambie)
>
> $psql bulmafactdb
> < /usr/share/bulmages/dbmodels/plugins/revf-pluginsyncbulmacont.sql
>
> 4.- Tienes que indicarle a la base de datos de bulmafact cual es la base de
> datos de contabilidad con la que se sincroniza
>
> $ psql bulmafactdb
> # update configuracion set valor='bulmacontdb' WHERE nombre
> = 'DataBaseContabilidad'
>
> Una vez hecho esto ya estará en marcha el plugin de sincronización.
>
>
> COSAS MUY A TENER EN CUENTA:
>        1.- Conviene que antes de poner en marcha esto tengas en bulmacont
> las
> siguientes cuentas creadas:
>                - 43000...0 (Cliente varios,  sino el plugin empezará a
> crearte cuentas con
> sólo 4 dígitos )
>                - 44000...0 (Proveedor varios, sino el plugin empezará
> creandote cuentas de
> sólo 4 digitos)
>                - 472....16
>                - 472.....7
>                -472 .....4 (tantas de iva soportado como vayas a utilizar
> sino  al utilizar
> un iva que no tiene cuenta el programa dara un error)
>
>                - 477 .....XX (Lo mismo para el iva repercutido)
>                - 7000....00 (La cuenta de ventas para que las facturas
> vayan bien)
>
>                - Si usas recargos de equivalencia tienes que hacer cuentas
> del tipo
> 47201...16, 47201......4 para cada iva y lo mismo para las 477
>
>
>                - Seguro que me dejo alguna cuenta que hay que tener (Bancos
> o cajas)
>
>
>
>
> NOTAS:
>
>        1.- Hay un script de resincronización que es muy útil para
> sincronizar un
> sistema que ya está empezado. (no se donde esta instalado pero en el svn es
> facil de encontrar)
>        2.- Tambien tienes un script de desinstalación por si las moscas.
>        3.- Es muy útil leer el revf-pluginsyncbulmacont.sql No es difícil
> de
> entender y si quieres hacer asientos especificos, o diferentes a los que se
> estan haciendo lo puedes modificar.
