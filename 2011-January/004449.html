* aishen [19/01/11 08:06]:
> I have same problem on ubuntu it looks for qmake but ubuntu is qmake-qt4
> so i made a link : ln -s /usr/bin/qmake /usr/bin/qmake-qt4 and it works
> but it's only a work around

Thanks, 
this seems to be the culprit, but it is strange, first from what Tomeu
says that the compilation of Bulmages does not use qmake at all, and
second that there actually is a qmake in Ubuntu 10.10:

lrwxrwxrwx 1 root root 23 2010-07-23 06:30 /usr/bin/qmake -> /etc/alternatives/qmake*

and it is a link to /etc/alternatives/qmake

which in turn

lrwxrwxrwx 1 root root 18 2010-07-23 06:30 /etc/alternatives/qmake -> /usr/bin/qmake-qt3*

is a link to /usr/bin/qmake-qt3

But there is also:

/usr/bin/qmake-qt4

so, quite confusing. I will changed the symbolic link of qmake
to qmake-qt4 and you are right, now "cmake ../" completes without
error and with "make" compilation started as usual .
