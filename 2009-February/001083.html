> > El Friday 13 February 2009 20:41:26 Josep Maria EstopÃ escribió:
> >> Hola de Nuevo,
> >>
> >> ahora que ya he visto que me funciona la lista voy a dar mis primeras
> >> impresiones sobre el TPV, comentar lo que considero son bugs o fallos
> >> y posibles mejoras que creo que serían muy útilies.
> >>
> >> El TPV está instalado en un bar, en un PC pentium 4 a 2,6Ghz y 1GB de
> >> Ram (más que de sobra para el TPV). La pantalla es un TFT de 15'' y no
> >> es táctil. Lleva el cajón con el dinero y una impresora matricial de
> >> tickets por puerto paralelo. El ordenador lleva una Ubuntu 8.10 con
> >> Gnome.
> >>
> >> Las primeras sensaciones con el TPV son buenas, funciona bien en
> >> general y cumple con su trabajo  (buen chico ;-) ). No voy a hablar
> >> sobre la instalación porque no la hice yo mismo y bueno no dió
> >> problemas. Sólo comentar que se instaló a partir de los fuentes
> >> compilando en el propio ordenador.
> >>
> >> He dado de alta los artículos en BulmaFact y he creado las plantillas
> >> para las pantallas de productos del TPV y he cargado las imágenes de
> >> los productos del TPV que tenía anteriormente.
> >>
> >> Fallos, lo que se dice fallos, solo he detectado tres cosas:
> >> 1. La impresión de la X (resumen temporal de los movimientos del TPV
> >> desde la última Z) no saca los importes, sólo la cantidad de tickets.
> >> 2. La impresión del ticket de la X y de la Z no me salen bien en la
> >> impresora de Tickets, le faltan columnas. Mientras no sea configurable
> >> creo que debería usar el mínimo de columnas necesario para mostrar esa
> >> información.
> >> 3. El plugin para el cambio, al cobrar, no tiene botón para la coma
> >> decimal. Creo que más que poner un botón para la coma bastaría con que
> >> considerara los últimos dos dígitos como decimales (siempre que demos
> >> por hecho que trabajamos en Euros, luego vuelvo sobre el tema de la
> >> moneda). A nivel de edición la pantalla debería mostrar algo así:
> >> - cuando introduces un 2 debería mostrar 0.02, cuado introduces un 20,
> >> debería mostrar 0.20, cuando introduces 200 deberia mostrar 2.00.
> >>
> >> Dicho esto, voy a comentar posibles mejoras que me parecen importantes
> >> para el TPV y luego para el BulmaFact.
> >>
> >> BULMATPV
> >>
> >> - En la pantalla en la que se van viendo las lineas del ticket sería
> >> interesante que tuviera un botón de eliminar el ticket completamente,
> >> estoy hablando siempre de tickets abiertos, que aun no tienen núero de
> >> ticket y con un mensaje de confirmación.
> >>
> >> - En un ticket vacío cunado se pulsa el botón de borrar línea sale un
> >> mensaje avisando que el ticket está vacío. Creo que sobra. Como mucho
> >> pondría emitiría algún sonido o algún efecto visual como un parpadeo o
> >> un cambio de color en el ticket para indicar que está vacio, pero nada
> >> que bloquee la pantalla.
> >>
> >> - Poder recuperar un ticket en el TPV para revisarlo (NO MODIFICARLO)
> >> y reimprimirlo, y que muestre el literal "es copia" (opcionalmente que
> >> se pueda quitar ese literal). Es interesante porque si coincide el
> >> final del rollo de papel con el ticket y ya se ha cobrado ya no se
> >> puede repetir.
> >>
> >> - Poder anular un ticket desde el TPV, que no significa borrarlo,
> >> simplemente marcarlo como anulado. Esto es porque no se puede
> >> legalmente eliminar un ticket, no pueden haber saltos entre tickets y
> >> se debe poder justificar cuando falta uno.
> >>
> >> - Poder configurar el TPV para que sólo imprima los tickets bajo
> >> demanda o siempre, según se quiera. Es muy interesante para el ahorro
> >> de papel y mas en un lugar donde la gente no suele pedir el ticket.
> >>
> >> - Previsuaización por pantalla de la X y la Z con opción de imprimirlas
> >> o no.
> >>
> >>
> >> BULMAFACT
> >>
> >> - Al dar de alta los arttículos estaría bien (y en otros maestros
> >> tambien) que cuando se trata de una alta (no una modificación) cuando
> >> se ha grabado el artículo se volviera a mostrar la pantalla de alta,
> >> lista para introducir el siguiente artículo.
> >>
> >> - Para negocios pequeños iría bien que se pudieran establecer algunos
> >> valores por defecto, como familia, tipo artículo y tipo de IVA, para
> >> no tener que introducirlos cada vez, o que recuerden el del último
> >> introducido.
> >>
> >> - Respecto al código de artículo sería interesante que se pudiera
> >> definir si se desea usar un código numerico y que el programa nos
> >> diera el siguiente código en función de la familia que estamos
> >> introduciendo.
> >>
> >> - Algo mas sencillo, creo, sería interesante que al pulsar INTRO/ENTER
> >> hiciera lo mismo que e botón ACEPTAR, eso agilizaría la introducción
> >> de datos.
> >>
> >> - Yo uso el plugin de IVA incluido, por la cual cosa el campo de
> >> precio del artículo es un plugin. Con el tabulador no se puede llegar
> >> al campo del precio tienes que hacerlo con el ratón, lo cual entorpece
> >> un poco la introducción de datos. Intuyo que es un problema de los
> >> plugins y que se debería plantear una solución a nivel global para
> >> todas pantallas que usen o sean susceptibles de tener plugins.
> >>
> >>
> >> Creo que esto es todo por el momento. Iré comentando lo que vaya
> >> viendo en el día a día.
> >>
> >> Un cordial saludo,
