Relacion de cambios:

* Renombro 'plugintecladonumerico' a 'pluginbt_tecladonumerico'.

* Se han aplicado una serie de cambios que se pueden extender al resto de plugins

pluginbf_tecladonumerico:
--------------------------
Se han creado los directorios
'css' -> Se guardan las hojas de estilo especificas para ese plugin.
'images' -> Se guardan los archivos de imagenes que las hojas de estilo utilicen.

La idea es disponer de una configuracion 'css' general para cada proyecto
dentro de '/installbulmages/css': 'bulmages.css', 'bulmacont.css', 'bulmafact.css' y 'bulmatpv.css'

Cada plugin tendra un archivo 'css' si es necesario (no es obligatorio).

A la hora de compilar:
----------------------
Si el archivo 'css' del plugin existe se fusionara (APPEND) automaticamente con el archivo general de su proyecto.
Por ejemplo, en el caso de 'pluginbf_tecladonumerico' se fusionara con 'bulmatpv.css' (si el archivo no existe se crea).
Para lograr esto hay que modificar el 'CMakeList.txt' del 'plugin' con:

-- INICIO CODIGO --

configure_file( ${CMAKE_CURRENT_SOURCE_DIR}/css/${plugin}.css.in ${plugin}.css @ONLY)

FILE(READ ${CMAKE_BINARY_DIR}/${plugin}.css VAR_CSS)
FILE(APPEND ${CMAKE_BINARY_DIR}/css/bulmatpv.css ${VAR_CSS} )

install(DIRECTORY images DESTINATION share/bulmages PATTERN ".svn" EXCLUDE)

-- FIN CODIGO --

Esta ultima linea copia los archivos de imagenes en el proceso de instalacion.

Para no tener conflictos con los nombres de los archivos de otros plugins hay que seguir la guia de estilo.
En el caso de este plugin cada imagen tiene como prefijo el nombre del plugin 'pluginbt_tecladonumerico'.
De esta manera siempre se sabra a que 'plugin' corresponde cada imagen.

(Para definir la hoja de estilo a usar se utiliza la directiva de configuracion CONF_STYLESHEET)

AVISO: Repetir el proceso de 'cmake' repetidas veces sin borrar hace crecer los archivo 'css' por el proceso de mezcla (APPEND). Esto no tiene mayores consecuencias.

* Con este 'plugin' se adjuntan unas imagenes de las que soy autor y licencio bajo la GPL v2 o superior.
