Para crear un nuevo plugin, se crea como se han ido creando el resto de 
plugins. Los pasos son los de siempre:

- mkdir dir_nombre_del_plugin
- crear los ficheros que toquen
- Copiar el CMakeLists.txt.template i cambiar poner el nombre que toque.
- crear el directorio po dentro del del plugin.

Ir a build, hacer make
Si todo ha ido bien entonces hay que haver:
- make nombre_del_plugin_pot
(esto creará el template de mensajes)
- make update_nombre_del_plugin_pot
(esto copiará el template al source_dir)

Ahora hace falta inicializar los mensajes. Dentro del directorio po, donde 
reside el fichero nombre_del_plugin.pot ejecutar el script que se encuentra 
en build.
~/donde_este_bulmages_build/create_gettext_files nombre_del_plugin

este script creará y actualizará los mensajes a partir de los catálogos. Por 
defecto, crea la versión castellana y la catalana. Si queremos realizar 
alguna traducción, corrección, etc, podemos editar el fichero creado.

Si todo ha ido bien, ya se  puede hacer:
make translations
make install

Si, creemos el el plugin ya está esta con mensajes correctos, entonces 
deberiamos actualizar el catálogo de mensajes:

Dentro de build, ejecutamos:
./create_message_catalog

donde nos creará un nuevo fichero pot, con todos los mensajes de la suite 
bulmages y lo colocará en los sources.

y luego,

./update_message_catalog_lang language1 language2 language3

por ejemplo:

./update_message_catalog_lang ca_ES.UTF-8 es_ES.UTF-8 

donde, a partir de los mensajes (incluso los que acabamos de editar), 
actualiza el catálogo de mensajes traducidos de sources.

La idea es que se vaya traduciendo estos ficheros (los de dentro de 
installbulmages/i18n)

Cuando queramos, podemos actualizar los ficheros de traducción de cada módulo, 
simplemente ejecutando:
make update_lang_po

Ir con cuidado porque esto actualiza unos 200 ficheros.

Bueno, ahora solo falta probar, depurar y traducir. Convendría tener algun 
tipo de convención para los mensajes y no tener mensajes gloriosos ....

> del plugin y comentar las siguientes líneas de GETTEXT_CREATE_TRANSLATIONS:
>
> if(USE_GETTEXT)
>    set(plugin_i18n ${plugin_SRC} ${plugin_MOC_SRCS} ${plugin_UIS_H}
>                         ${plugin_SRCS} ${plugin_MOCS})
>    GETTEXT_CREATE_TEMPLATE(${plugin} ${CMAKE_CURRENT_BINARY_DIR}
>                                 ${CMAKE_CURRENT_SOURCE_DIR} plugin_i18n)
> #   GETTEXT_CREATE_TRANSLATIONS(${CMAKE_CURRENT_BINARY_DIR}/${plugin}.pot
> #                                ALL LANGUAGES es_ES.UTF-8 ca_ES.UTF-8)
> endif(USE_GETTEXT)
>
>
> 2)- cmake (como siempre)
> 3)- make
> 4)- make translations
> 5)- entrar en el directorio del plugin en el directorio de compilación y
> ejecutar 'make pluginbf_trabajador_pot' para generar el archivo '.pot'
> con las cadenas a traducir.
> 6)- crear directorio 'po' en el directorio del plugin (código fuente).
> 7)- copiar el archivo '.pot' desde el directorio de compilación.
> 8)- copiar el archivo '.pot' para cada idioma con extensión '.po'. Ej:
> pluginbf_trabajador_es_ES.UTF-8.po
> 9)- descomentar las líneas de CMakeLists.txt
> 10)- cambiar, de los archivos '.po' las líneas 16 ..." charser=CHARSET"
> por "charset=UTF-8" porque sino dará error en el siguiente paso.
> 11)- make translations (otra vez. esto traducirá las cadenas que pueda
> de forma automática, rellenando los archivos '.po' de cada idioma)
> 12)- copiar los archivos '.po' del directorio de compilación (que ya
> está traducido) al directorio del código fuente del plugin.
