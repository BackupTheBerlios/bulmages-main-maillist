Prosigamos con las enseñanzas de GIT, vamos a manejarnos entre las
ramas de una manera sencilla.



* Se ha reportado un bug en la rama estable y queremos resolverlo.

Actualizamos como siempre nuestro código

0.) git pull



Nos trasladamos a la rama estable y la convertimos en una rama local
para poder hacer los cambios

1. ) git checkout origin/estable

2.) Creamos una nueva rama, en base a la actual con el nombre estable-fix.

2.1) git checkout -b estable-fix

3.) Editamos, arreglamos el bug, y confirmamos que esta solucionado.

4.) git commit -a

5.) git push



* Hemos solucionado el bug, y queremos ir a otra rama, recordemos que
la rama de desarrollo es la "master".

1.) git checkout <RAMA_DESEADA>
