*** DocBook ***

DocBook is a schema maintained by the DocBook Technical Committee of OASIS. It is particularly well suited to books and papers about computer hardware and software (though it is by no means limited to these applications). 

Según comentan, es el formato de documentación oficial de KDE, GNOME, del kernel de Linux, etc, etc.

Como DocBook es un xml: no describe la apariencia final del documento, sino su contenido.

DocBook tiene diversas ventajas, entre las cuales tenemos:
 - Se parte de un archivo XML (que se puede crear con un editor de texto o mediante herramientas especializadas.) DocBook señala el conjunto de tags válidos, estructura y significado, a fin de componer documentos técnicos de manera estandarizada.
 - Se puede generar una versión del documento en el formato más apropiado (en este caso se disculte el PDF)
 - Permite las facilidades de indexación propias de un documento estructurado con XML

En (K)ubuntu para hacer algunas transformaciones a formato html se pueden instalar los paquetes docbook-xml , xsltproc y docbook-xsl

+++ Conversión a formato html +++

Se puede crear un archivo "test.docbook" que contenga

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE book PUBLIC "-//OASIS//DTD DocBook XML V4.4//EN"
  "/usr/share/xml/docbook/schema/dtd/4.4/docbookx.dtd">
 
<book>
 
  <bookinfo>
    <title>Hello, world</title>
  </bookinfo>
  
  <chapter>
    <title>Hello, world</title>
  
    <para>This is my first DocBook file.</para>
  
  </chapter>
</book>

y al final ejecutar 
  xsltproc -o resultado.html /usr/share/xml/docbook/stylesheet/nwalsh/xhtml/docbook.xsl test.docbook
y abrir el archivo resultante. 

Más información en http://opensource.bureau-cornavin.com/crash-course/en/hello-world.html


En https://help.ubuntu.com/community/DocBookEditors se decía:

    5.4. Customising style sheets

    You will probably be disappointed with the look of the DocBook output. It is great to have it convert the document automatically but it probably does not fit into your web design or 'corporate identity' at all. There is a remedy however.

    Style sheets usually provide a number of parameters that you can adjust. Usually you write your own stylesheet that imports the 'standard' style sheet. For example::

    <?xml version='1.0'?>
    <xsl:stylesheet xmlns:xsl="http://www.w3.org/1999/XSL/Transform" version="1.0">
        <xsl:import href="/usr/share/xml/docbook/stylesheet/nwalsh/xhtml/docbook.xsl"/>
        <xsl:param name="toc.max.depth">1</xsl:param>
        <xsl:param name="html.stylesheet" select="'/ubuntu.css'"/>
        <xsl:template name="user.header.content">
          <a href="/">Back to main page</a>
        </xsl:template>
    </xsl:stylesheet>

    This stylesheet first imports the docbook.xsl mentioned earlier. It also sets a few parameters:

    - Set the maximum depth of the TOC (table of contents) to '1'. So only the <sect1> sections will be included in the TOC. 

    - The final XHTML document will use the 'ubuntu.css' style sheet (CSS). - Include a link to the main page on top of the page.

    These settings only work with the XHTML style sheet. For other output format you need other settings. The settings above are documented at /usr/share/doc/docbook-xsl/doc/html/index.html

    You will also want to use http://www.sagehill.net/docbookxsl/ as a reference.

    5.5. Makefile

    If you have multiple XML files or style sheets you may want to have all the processing done in a Makefile. For example::

    # Add your language file here:
    TARGETS = faq.html

    XSLTPROC = /usr/bin/xsltproc
    XSL = ubuntu.xsl

    %.html: %.xml $(XSL)
        @$(XSLTPROC) -o $@ $(XSL) $<

    all: $(TARGETS)

    clean:
        @rm -f *.html



+++ Conversión a formato pdf +++

En http://americati.com/blog/?p=38 se decía:

  [...] el "Apache fop"
    fop -xml prueba.xml -xsl "/usr/share/xml/docbook/stylesheet/nwalsh/fo/docbook.xsl" -pdf prueba.pdf

  Este comando asume que nuestro documento ?prueba.xml? sigue las reglas de DocBook, y deseamos generar una versión PDF. Asimismo estamos empleando el archivo XSL distribuido con DocBook para transformar el documento XML a un formato intermedio (llamado ?fo? o ?formatting-objects?) el cual se emplea para la generación de formatos gráficos (PDF entre ellos.) En otras palabras, este proceso involucra dos pasos, los cuales también se pueden hacer con la siguiente secuencia:

    xsltproc --output prueba.fo /usr/share/xml/docbook/stylesheet/nwalsh/fo/docbook.xsl prueba.xml

    fop prueba.fo -pdf prueba.pdf

  Esta segunda forma sorprendentemente es mucho más veloz pues el primer paso es casi inmediato
