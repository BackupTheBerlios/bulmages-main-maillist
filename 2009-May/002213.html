El 26/05/09, Arturo Martín Lladó <amartin en conetxia.com> escribió:
> Espero que esta explicación te sirva.
>
> El usuario "root" de PostgreSQL es el usuario de sistema "postgres".
> Éste es el superusuario de la base de datos con más privilegios.
> Después, postgres puede añadir usuarios y ponerlos como superusuario o
> no (y creo que aún así tienen menos privilegios de administración en
> algunos detalles).
>
> A lo mejor te está liando que, desde bulmasetup, se añada al root como
> superusuario de PostgreSQL. Esto es porque, para realizar las conexiones
> a la base de datos es necesario que este usuario esté creado, si no,
> bulmasetup no puede realizar conexiones a las bases de datos para
> administrarlas, ni administrar usuarios, etc.
>
> Recuerda que bulmasetup se ha de ejecutar como root para que pueda
> funcionar, así que, usando la autenticación ident sameuser (o trust que
> no sé si va como predeterminada) de PostgreSQL, se conecta a la base de
> datos con el "same user" de UNIX, en este caso, root.
>
> A lo mejor esto también ayuda:
> http://wiki.postgresql.org/wiki/Autenticaci%C3%B3n_del_Cliente
>
> Un saludo.
