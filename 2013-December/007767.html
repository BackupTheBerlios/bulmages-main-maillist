En el ejemplo que has puesto esta claro que se pretende favorecer al C++ y
deberían retirarle el carnet de farsante por hacerlo tan mal (no usa
encapsulación adrede). El ejemplo en ambos lenguajes se resuelve con una
línea de código.

Creo que cuando aconsejaron empezar por C se referían a que usase C++ sin
orientación a objetos y luego fuese incorporando nuevos conceptos. Si no
sabes las bases de C es imposible que puedas aprender el paradigma
orientado a objetos. (Igual que construir una casa empezando por el tejado).

> ¡Hola!

> > Pienso que primero deberías empezar por C,
>
> Sólo quería comentar que [dice Stroustrup]
> (http://www.informit.com/articles/article.aspx?p=2080042)
> que es mejor aprender primero la programación orientada a objetos, que
> simplifica más las cosas, etc. Por ejemplo está la diferencia entre el
> código
> de C++:
>     string compose(const string& name, const string& domain)
>     {
>         return name+’@’+domain;
>     }
>    [...]
>    string addr = compose(“bs”,”cs.tamu.edu”);
>
> y su versión no orientada a objetos, en C:
>
>     char* compose(const char* name, const char* domain)
>     {
>         char* res = calloc(strlen(name)+strlen(domain)+2);  // space for
> strings, ‘@’, and 0
>         char* p = strcpy(name,res);
>         *p = ‘@’;
>         strcpy(domain,p+1);
>         return res;
>     }
>     [...]
>     char* addr = compose(“bs”,"cs.tamu.edu”);
>     // ...
>     free(addr); // release memory when done
>
>
