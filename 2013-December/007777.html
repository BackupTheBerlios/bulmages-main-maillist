Por si alguien tiene curiosidad, en el std::cout de iostream, lo equivalente a 
    printf("%10.2f", x);
podía ser    
    cout << setw(10) << setprecision(2) << showpoint << x;
que añadia claridad para quien estaba aprendiendo (se usa "set wide", "set 
precision", "show point"), no se ha de preocupar de tener que especificar el 
tipo de dato a imprimir, no hay equivocación al especificar el tipo de dato ya 
que no se especifica, etc. 

Usando otras clases C++, como QString de Qt y QTextStream (que en Qt te 
permite crear un objeto "QTextStream cout(stdout)" y otros con más 
funcionalidades):
1) Lo equivalente a
    printf("%10.2f", x);
puede ser 
    cout << QString().sprintf("%10.2f", x);
lo que tiene sus ventajas :-) a la hora de adaptar un programa antiguo, por 
poner un ejemplo. 

2) Si se escribe 
    cout << QString("El número es: %1").arg(x);
Alguien puede plantearse, ¿y "arg()" no me permitirá especificar la cantidad de 
decimales a escribir y otras características del formato? Efectivamente:
    QString::arg(double a, int fieldWidth = 0, char format = 'g', 
                 int precision = -1, QChar fillChar = QLatin1Char( ' ' ))
    QString::arg(int a, int fieldWidth = 0, int base = 10, 
                 QChar fillChar = QLatin1Char( ' ' ))
    [...]
Se permite especificar el formato de una forma más clara a primera vista, lo 
que viene muy bien, como mínimo cuando se está aprendiendo.
