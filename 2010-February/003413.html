Log:
En la FUNCTION syncbulmacontfacturau ():
- Protejo las ejecuciones de dblink dentro de su propia transacción. Se
  implementa que en caso de error en la transacción de la función se
hacer primero un ROLLBACK de la conexión dblink.
- se cambia 'IF NOT FOUND THEN' por 'IF <loquetoque> IS NULL THEN' en
  las consultas que siempre devuelven una fila aunque sea NULL y que no
se detectan bien al usar 'IF NOT FOUND THEN'. Esto provocaba fallos al
no establecerse adecuadamente los valores por defecto en ciertas
variables.
- Se corrige la gestión de apunte de IVA y Recargo de equivalencia.
  Pasaba que se registraban varios apuntes para un mismo tipo de IVA si
existían líneas en la facturación con recargo de equivalencia. Ahora se
agrupan y se evita un fallo de desviación de cálculo de IVA (de
céntimos).

Se crea la gestión automática de cuentas para las familias. Al crear una
nueva familia en BulmaFact se crean 2 cuentas automáticas en la
contabilidad, una para VENTAS y otra para COMPRAS. Queda para una
próxima revisión el establecimiento manual de cuentas para las familias
de artículos.
