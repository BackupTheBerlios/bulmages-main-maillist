 Hemos limpiado y dividido los directorios en distintos repositorios git, con la siguiente infraestructura.

    *  Proyecto central - Bulmages ( 
https://gitorious.org/bulmages/bulmages     *  ) de 107,5 MB
    *  Proyectos separados Bulmages-Extra ( 
https://gitorious.org/bulmages-extra     *  )
 Dentro de los subproyectos nos encontramos con:

    *  Archivos para el empaquetado Debian (pendiente de revisión) (
https://gitorious.org/bulmages-extra/debian-packaging     * ) de solo 568,6 kB
    *  Archivos para la creación de el instalador Windows con NSIS (con su rama nsis_fapacges) ( 
https://gitorious.org/bulmages-extra/nsis_bulmages     *  ) de 562,8 MB
    *  No-Cont-Bulmages, proyecto web que debía acceder a bulmacont (obsoleto) (
https://gitorious.org/bulmages-extra/no-cont-bulmages     * ) de 13,0 MB
    *  Bulmages-nds, proyecto de TPV para NDS iniciado por Francisco Javier de Todo-Redes (
https://gitorious.org/bulmages-extra/bulmages-nds     * ) de 123,8 MB
 Con esta nueva estructura conseguimos unas ventajas muy importantes:

    *  Reducción de peso del repositorio central, por lo tanto más velocidad a la hora de desplegar el proyecto
    *  Eliminados los directorios obsoletos o pendientes de revisión que no tenian nada que ver con el proyecto principal.
    *  Al ser más reducido, los programadores o usuarios podrán descargarse solo lo que les interese sin arrastrar el bloque entero.
    *  Mayor belleza estetica. (lo de menos)

https://gitorious.org/bulmages-extra/debian-packaging
