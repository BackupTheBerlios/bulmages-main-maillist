Hay otra diferencia.

El DbRequired comprueba que el campo este relleno antes de agregar una nueva
linea en el subformulario (caso de los subformularios con la fila vacia para
rellenar una nueva y que sólo aparece la fila nueva cuando has completado la
fila anterior. La comprobación es que todos los campos DbRequired tengan
contenido) mientras que el DbNotNull sólo indica que en la base de datos se
producirá un error si el campo está vacio.

Por tanto uno es para ayudar a la introducción de datos y el otro es para la
integridad de datos y evitar el mensaje de error.

> Al definir los datos de un listado BlSubForm tenemos dos opciones para
> campos "NOT NULL": BlDbField::DbRequired y BlDbField::DbNotNull.
>
> En ambos casos, si el valor es nulo, no se realiza el INSERT o UPDATE,
> aunque DbNotNull muestra un error y DbRequired no lo hace. ¿Es la única
> diferencia?.
>
> Un campo que no puede ser nulo podría tener un valor predeterminado
> definido
> en la base de datos, sólo habría que evitar nombrar al campo en el INSERT o
> UPDATE. En tal caso, si el comportamiento actual es el deseado, habría que
> agregar una nueva posibilidad: "BlDbField::DbIgnoreIfNull".
>
> Y otro detalle: cuando se evita realizar el INSERT o UPDATE, se pierden los
