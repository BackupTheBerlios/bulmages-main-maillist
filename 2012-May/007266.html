Hola,
Con el ultimo commit acabo de fixear el problema que dijo Tomeu sobre 
que caba error al ejecutar CMakes antiguos,
si se pega una ojeada al CMakeLists.txt principal, veréis que compruba 
si es una versión posterior de CMAKE a 2.8.1 (cuando se introdujo 
CMAKE_PARSE_ARGUMENTS), en caso contrario intenta buscar python mediante 
un IF(EXISTS) distinto para unix y para win32 y luego lo guarda en la 
misma variable cache de cadena PYTHON_EXECUTABLE de esta manera, solo 
nos hemos de aprender @PYTHON_EXECUTABLE para hacer referencia a la ruta 
de python.

Obviamente, pueden añadirse nuevas rutas a mano.
