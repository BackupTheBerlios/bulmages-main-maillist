Log:
Small hack to allow accented words to appear correctly in the 
select box for plugin categories in bulmasetup. Now localized 
categories in the "descripcion" field of the README.ca of the 
plugins may have non-ASCII characters, such as "Facturaci?\195?\179
avan?\195?\167ada". This may be fixed somewhere else as it seems strange
that this is the only place in bulmasetup where the program 
wants iso-8859-1 characters (for instance if the README.ca files
are coded iso-8859-1 then the characters appear correctly in the
select box but incorrectly everywhere else).
