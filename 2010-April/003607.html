Log:
Cambio el tratamiento del punto fijo para que la precision por defecto en un resultado sea la mayor de las precisiones de los operandos. También altero los subformularios para que la precision de sus campos tipo DBNumeric sea la misma que tiene el campo en la base de datos con lo que las precisiones en lugar de fijarse a dos vienen indicadas por la que tiene puesta la base de datos. NOTA: Para alterar la precision de un campo en la base de datos hay que usar pgadmin3 o la formula ALTER TABLE table ALTER columna TYPE numeric(12,precision);
