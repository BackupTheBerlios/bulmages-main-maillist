Muy bien !!!

Solo puntualizar que no se crea un archivo nuevo de parches para cada cambio 
sino que hay un único archivo de parches por versión que contiene todos los 
cambios para esa versión. Y como tiene la carácterística de no reaplicar los 
cambios que ya han sido realizados se puede aplicar tantas veces como se 
quiera sobre una base de datos porque no la deja en un estado lamentable tras 
multiples aplicaciones del mismo.

Y ya que estamos agregar que los parches para bulmacont son los que empiezan 
por rev y los de bulmafact los que empiezan por revf

No hay parches para bulmatpv ya que bulmatpv funciona con la misma base de 
datos que bulmafact (A la que previamente se le ha aplicado un parche). Que 
está en el directorio bulmatpv.

El Tuesday 18 November 2008 16:28:24 Arturo Martín Lladó escribió:
> Comentar también que cuando se modifica el esquema de base de datos, ya
> sea en BulmaFact o en BulmaCont el procedimiento es el siguiente:
>
> 1. Se modifica el SQL con el esquema de la base de datos.
> 2. Se crea un parche que aplica los cambios a una base de datos en
> producción.
>
> Así hay dos maneras de disfrutar los nuevos cambios, creando una nueva
> base de datos (se usa el archivo mencionado en 1) o parcheando la ya
> existente (usando el parche SQL mencionado en 2).
>
> Los SQL se guardan en trunk/bulmages/installbulmages/dbmodels
>
> - El subdirectorio actualizar contiene los parches para bulmafact y
> bulmacont.
> - El subdirectorio crear contiene los SQL con el esquema completo para
> bulmafact y bulmacont.
> - El subdirectorio plugins contiene los SQL de los plugins que para
> usarse necesitan parchear la base de datos.
>
> Espero que haya servido este texto.
>
> Un saludo.
