Hola a tothom:
Yo lo haría de esta forma (que no es como lo hago pero creo que es el método 
más asequible a un usuario que no quiere liarse con el código):

$ svn update
$ cd build (o mkdir build si no existe ya)
$ cmake ../ (por si hay plugins nuevos y hay que refrescar los CMakeList)
$ sudo make install (Como root y que se instale en el /usr/local por defecto)

Luego

$ sudo bulmasetup

Modificar / Actualizar empresas
Abrir una a una y darle al boton "Actualizar Base de Datos" (Si hay parche se 
aplica y si no no hace nada).

  Si quieres ahorrarte este último paso hay que seguir los mensajes de commit 
para ver si hay cambios en los archivos sql o no.


NOTA: Es importante saber que los archivos genericos de configuración se 
sobreescribiran incorporando los nuevos parametros que se hayan agregado. Sin 
embargo los archivos de configuración de cada empresa particular quedarán 
intactos. Así que si eres listo lo que harás es trasladar todos los cambios 
de configuración a los archivos específicos de empresa para que no se pierdan 
cada vez que actualizas.

NOTA2: Parece más de lo que és, pero en realidad es muy sencillo.

Salut
