Este problema es debido a la forma de multiples archivos de configuracion. 
Para mi que lo mas correcto seria dejar el bulmafact.conf sin ningun plugin 
por defecto.

Me explico:
Bulmages permite tener una configuración distinta para cada empresa creada 
(esto es tener varias empresas usando plugins distintos)
Para ello cada empresa puede tener su propio archivo de configuracion 
bulmafact_XXXXX.conf  donde XXXXX es el nombre de la base de datos.
Bajo esta configuracion el programa lee primero el archivo bulmafact.conf y 
luego lee el archivo bulmafact_XXXXX.conf reemplazando las opciones leidas 
previamente con las que encuentra en este fichero. Asi la configuración de 
bulmafact.conf es generica para todas las empresas y la de 
bulmafact_XXXXX.conf especifica para cada emrpesa.

BulmaSetup no actua sobre bulmafact.conf sino que únicamente escirbe el 
archvio bulmafact_XXXXX.conf de ahi que si nunca habías generado la 
configuracion especifica se cogieran las configuraciones por defecto.

Ese es básicamente el problema. Luego hay más archivos de configuracion 
especificos para cada usuario pero no suele ser necesario usarlos.
