Buenos dias Manuel.

Las ultimas versiones para Linux son practicamente "portables". Cada una
instala todas sus dependencias (postgresql, qt, python ...) Lo preferimos
así pq si para poner en marcha el programa hay que instalar primero todas
las dependencias se puede convertir en un calvario. Eso hace engordar el
programa pero sigue siendo mejor que obligar a instalar postgresql y qt (y
luego la configuración también sería otro caos).

Para contribuciones en git lo mejor es crearse un clon y trabajar sobre
éste. Luego avisar de determinado cambio (git puede asustar pero a nivel
básico es sencillo).

También creo que una buena forma de contribuir es testeando, testeando y
probando el programa. Ya que en estos momentos esta en un buen nivel de
funcionalidades y la prioridad es enrobustecerlo.

Salut
