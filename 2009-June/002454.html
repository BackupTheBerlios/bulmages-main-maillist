Te cuento un poco como va el tema

Para las aplicaciones tenemos un "core" de base de datos que se aplica en toda 
empresa (uno para facturacion y otro para contabilidad). Ahora el core 
incluye muchas cosas pero en la proxima versión la idea es delegar a los 
plugins muchas de las tablas del core.

Dicho core puede cambiar de una versión a otra y para mantener las bases de 
datos actualizadas creamos unos parches que hacen la migración de una versión 
a otra.

Todo esto esta en installbulmages/dbmodels

Sobre los parches decir que tienen como requisito el ser transaccionales, no 
aplicables si la base de datos no esta en la version anterior o en la actual 
y que se pueden aplicar tantas veces como se desee sin incurrir en corrupcion 
de datos. De ahi que si abres uno tiene un poco de rollo en comprobaciones.

Y ahora la pregunta:

Cuando le das a aplicar cambios en la base de datos bulmasetup aplica los 
parches de actualizacion, en orden, para dejar la base de datos desde 
cualquier version a la actual.

Esta es la historia.
