>> La factura electrónica española se compone de dos partes:
>>
>> 1.- Generar un documento y firmarlo electronicamente. Ese documento de 
>> factura debe contener al menos la siguiente información:
>>
>>     * Núm. Factura
>>     * Fecha expedición
>>     * Razón Social emisor y receptor
>>     * NIF emisor y “receptor”
>>     * Domicilio emisor y receptor
>>     * Descripción de las operaciones (base imponible)
>>     * Tipo impositivo
>>     * Cuota tributaria
>>     * Fecha prestación del servicio (si distinta a expedición)
>>       http://www.facturae.es/Aspectos/Requisitos/
>>
>> en defintiva, la típica, la que ya se incluye en todas las facturas.
>>
>> 2.- Además, para una segunda fase, embeber la factura en un formato XML:
>> http://www.facturae.es/Documentacion/EsquemaFormato/Version3_1/
>> Con el XML se generaría un gran avance en la gestión de la 
>> contabilidad, ya que el procesamiento de facturas sería practicamente 
>> automático. La interoperabilidad entre sistemas sería del 99%, ya que 
>> la factura la emite un proveedor y la procesa un cliente. En fin, qué 
>> decir de las ventajas del uso de estandáres en esta lista...
>>
>> En definitiva, como factura electrónica valdría con cumplir el primer 
>> requisito, pero si además lleva el 2º lo bordamos.
>>
