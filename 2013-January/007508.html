Hola a todos,

Después de tanta documentación, automatización, git y demás mis paquetes
se pueden bajar directamente de:

	http://packages.mtelleria.be/bulmages/

Entiendo este repositorio como temporal con la intención de que se
migrase a la infraestructura bulmages (bulmages.es) si así lo aceptáis
en el proyecto.

Desde Debian se puede acceder a los paquetes con APT mediante las líneas
de /etc/apt/sources.list siguientes.

En Debian sid (y también Wheezy):

   deb http://packages.mtelleria.be/bulmages sid/
   deb-src http://packages.mtelleria.be/bulmages sid/

En Debian Squeeze:

   deb http://packages.mtelleria.be/bulmages squeeze/
   deb-src http://packages.mtelleria.be/bulmages squeeze/

NOTAD el '/' al final ya que si no apt pide una parámetro extra.

Para los interesados, he montado el repositorio con mini-dinstall[*]
siguiendo el guión de nuestro bien "amado leader" (Debian Project
Leader) Stefano Zacchiroli.

http://upsilon.cc/~zack/blog/posts/2009/04/howto:_uploading_to_people.d.o_using_dput/

Hace falta un servidor con el paquete mini-dinstall instalado para que
rehaga los índices.   Para alojarlo en un servidor que no tenga debian
o mini-dinstall se me ocurre usar rsync con una copia en una
Debian local como master.

[*]  Estuve mirando reprepro bastante popular y muy útil para mezclar
     distribuciones variadas.  Sin embargo tiene la restricción de no
     admitir más de una versión de un paquete en cada índice[**] lo cual
     me parece importante para un repositorio de paquetes en desarrollo.

     [**] http://bugs.debian.org/cgi-bin/bugreport.cgi?bug=570623

Saludos,

	Miguel
