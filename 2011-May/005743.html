Buenas tardes,

El hash que usa GIT posee información del archivo, tal como posición
en el historial, rama, fecha, etc... De ahi a que la manera
"secuencial" de SVN, pierda sentido.

Si se quiere seguir opteniendo ese "numero de revisión", podría
optenerse combinando git log con wc así:

git log --pretty=oneline | wc -l

Aunque a mi personalmente me gusta más el modo "rolling"  de nombrar
paquetes, por ejemplo.

bulmafact-git-20110506

Teniendo ahí la fecha completa del paquete.

También de paso mencionar, que quizá sería interesante echar una
ojeada a git-buildpackage, para la creación de paquetes *.deb.

Si queréis pegarle una ojeada, aqui os dejo link al manual:

http://honk.sigxcpu.org/projects/git-buildpackage/manual-html/gbp.html
