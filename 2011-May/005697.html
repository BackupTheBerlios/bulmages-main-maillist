¿Feim servir el INSTALL_PREFIX i només sortiran menus quan el INSTALL_PREFIX

Adjunto traducción (resumida) dada la importancia del tema:

El problema de las entradas de menu (Esto es que al instalar bulmages no hay
menus para lanzarlo y se tiene que tirar de Alt+F2 o de consola). Para hacer
los menus hay que poner algunos arxivos en localizaciones concretas pero
algo impide ser puristas con el tema y obliga a usar rutas absolutas si se
quieren instalar los menus desde el CMake. Si se instalan desde los paquetes
no sucede porque los paquetes siempre instalan en rutas absolutas. Entonces
hay que tomar una decisión:

1.- Se crean las entradas con make install o sólo con los paquetes ?

2.- Si se crean las entradas con el make install es preciso usar rutas
absolutas y salirse del INSTALL_PREFIX así funcionará siempre o usar el
INSTALL_PREFIX y que sólo funcione en el caso de que el directorio de
instalacion sea /usr

Lo correctísimo es usar el INSTALL_PREFIX y poner las entradas en
/usr/local/share y hacer que KDE busque los menus también en este directorio
... claro que si uno cambia el directorio de instalación y lo hace en /opt
tampoco funcionará pero segun el estandar Unix el directorio adecuado es
/usr/local

Enfin una historia ....
