Bones a tothom:

Tengo en mi clon la rama winversion que incorpora el NSIS para Windows.

La generación es "sencilla":
- Descargar
- Instalar 100.000 cosas que hacen falta.
- Configurar (cmake-gui) con rutas relativas ".." (hay un readme que lo
explica).
- Hacer make install (La instalación no funcionará, pero es necesaria para
el siguiente paso).
- Generar el instalador desde el directorio NSIS.

Aqui funciona el bulmasetup y se configuran las empresas como en Linux con
lo que opera bastante bien. Hace backups y restaura desde bulmasetup y la
sobreinstalación no elimina las bases de datos con lo que reinstalando
encima y actualizando con bulmasetup es factible.

Todo encaminado a la generación de instaladores rápidos cada vez que hay
cambios. Es decir que los "clientes" no tengan que esperar 1000 años para
arreglar errores. (La pega es que el instalador pesa 250Megas).

La duda es la siguiente:

Siendo la diferencia con cualquier otra rama un directorio.
(nsis_bulmages), y estando en  Git y no en subversion, donde las ramas son
bastante diferentes. ¿Merece la pena que sea una rama separada?.
Pq este detalle nos obligará a hacer un winversion14 y toda la parafernalia
asociada a hacer merges parriba y pabajo mientras que teniendo el
directorio como "uno más" se adapta perfectamente sin molestar a nadie.
